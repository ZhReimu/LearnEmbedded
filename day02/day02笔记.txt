===================IO操作=====================
一、文件IO的基本概念
linux系统下，一切皆文件

在我们的磁盘里面，除了文件夹就是文件，文件有各种各样的格式
图片（jpeg，png，gif，bmp。。。。）
文档（txt，c,h,cpp,doc......）
可执行文件(exe,msl,elf.........)
压缩文件(rar,7z........)
文件就是磁盘上的一段空间内存，不同的文件，格式就不一样，也就是说文件装的数据是不一样的，在宏观上进行区分

文档 txt:格式文件装的都是字符
图片 png:格式文件装的都是颜色标识
exe	：格式文件装的都是代码
本质还是0和1

文件IO就是对文件进行操作，对内容进行增删改查

txt	媒体文件	文本文档

文件七大类：
-	普通文件
d	目录文件	文件夹
c	字符设备驱动文件   鼠标  键盘  显示器。。。
b	块设备驱动文件	硬盘  U盘。。。。。
l	链接文件
s	网络文件	网络通信
p	管道文件	进程通信

open函数
man 2 open

#include <sys/types.h>
#include <sys/stat.h>       
#include <fcntl.h>

int open(const char *pathname, int flags, mode_t mode);
参数1：   路径名或文件名
参数2：   文件操作权限

O_RDONLY   以只读方式打开文件
O_WRONLY   以只写方式打开文件
0_RDWR
O_CREAT		打开的文件不存在的话就自动创建该文件
O_TRUNC		原本的文件中有数据，就清空数据
O_APPEND	会把光标移动到文件末尾，追加写入数据



返回值  return value
打开成功：返回一个文件描述符
打开失败：返回一个-1；

#include <unistd.h>

ssize_t  write(int fd, const void *buf,size_t count);
参数1：文件描述符
参数2：写进去的数据的缓冲区
参数3：写入的字节数

返回值：
成功：返回写入的字节数
失败：-1；移动文件的读写位置



#include <unistd.h>

ssize_t read(int fd, void *buf, size_t count);
参数1：文件描述符
参数2：读取出来的数据保存到这个buf缓冲区，文件的当前读写位置向后移动
参数3：需要读取的字节数，count为0，read函数不起作用了
返回值：
读取成功：返回读取的字节数
读取失败：返回-1，并且设置成errno；

read函数之前，光标位置已经是文件末尾，返回0写入数据
